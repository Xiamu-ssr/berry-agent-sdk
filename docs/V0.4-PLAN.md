# Berry Agent SDK v0.4 — Crash Recovery Internal + Polish

**状态**: 进行中
**起始**: 2026-04-22 17:00
**上下文**: v1.4-v1.6 的 DURABILITY 三阶段已完工，event log + snapshot + tool crash 检测都有了。但 `Agent.fromLog()` 作为公开 API 是误设计 —— crash recovery 应该 SDK 内部消化，产品层无感。本轮把它下沉、加审计、清理、打包 v0.4。

## 核心原则

1. **SDK 边界**：crash 检测逻辑只在 core，observe 被动收集，berry-claw 零改动
2. **唯一事实源**：类型/枚举/检测逻辑单点定义，前后端只读
3. **不生硬新增**：crash recovery 融进现有 observe 结构（turns 表加字段 + tool_calls 复用），不新建 tab/表
4. **event log = source of truth**，SQLite 是派生视图
